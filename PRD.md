### **产品需求文档 (PRD): 青海地区老年子宫内膜癌智能诊疗及随访平台 V1.0**

**版本:** 1.0
**日期:** 2025年11月4日
**作者:** Gemini

### 1. 背景与目标

#### 1.1. 项目背景
根据《青海省科技计划项目合同书 (2025-QY-220)》，本项目旨在解决青海地区老年子宫内膜癌患者面临的医疗资源分布不均、数据样本量小、预后和康复评估手段有限等问题。通过AI技术，开发一个集数据管理、智能分析、临床辅助决策和康复随访于一体的综合性平台。

#### 1.2. 产品目标
*   **近期目标 (V1.0):**
    1.  **建立标准化的数据集：** 开发一套软件工具，支持多中心、多模态（MRI影像、病理、随访信息）临床数据的采集、标注和管理。
    2.  **实现核心模型功能：** 集成基础的AI模型，包括用于MRI病灶区域识别的U-Net模型和用于预后风险因素预测的线性回归模型。
    3.  **打造临床应用原型：** 开发一个医生可用的临床辅助诊疗平台，可视化展示患者数据和AI分析结果。
    4.  **构建智能随访系统：** 开发一个智能化的康复随访系统，支持远程随访和分级诊疗。
*   **远期愿景:** 将该平台推广至青海省内多家医院，提升区域内子宫内膜癌的整体诊疗水平，并为医工结合培养复合型人才。

#### 1.3. 目标用户 (用户画像)
*   **临床医生/科研人员 (王医生):**
    *   **角色:** 青海省第五人民医院的肿瘤科医生。
    *   **痛点:** 患者数据管理繁琐，依赖个人经验判断预后，患者术后随访联络困难，工作负担重。
    *   **期望:** 希望有一个工具能集中管理患者的所有资料，提供客观的预后评估建议，并能自动化管理患者的术后康复和随访。
*   **患者 (李阿姨):**
    *   **角色:** 68岁，居住在偏远地区，刚完成子宫内膜癌手术。
    *   **痛点:** 去大医院复查路途遥远、成本高；康复期间有很多疑问，但无法及时联系到主治医生。
    *   **期望:** 希望能有一个简单的方式在家就完成定期复查提醒、健康上报，并能方便地获得康复指导。
*   **算法工程师/研究生 (小张):**
    *   **角色:** 北京石油化工学院的研究人员。
    *   **痛点:** 获取高质量、已标注的医疗数据困难；模型训练和验证流程不规范。
    *   **期望:** 希望平台能提供清晰的数据标注工具和结构化的数据库，方便进行算法开发和模型验证。

### 2. 系统整体设计

#### 2.1. 系统架构
系统主要由四个核心模块组成，部署在云服务器上，通过Web浏览器访问。患者端可通过微信小程序或Web应用进行交互。



#### 2.2. 功能模块概览
1.  **数据采集与标注模块:** 负责病例数据的录入、标准化处理和AI辅助标注。
2.  **AI预后评估模块:** 后端模块，负责运行U-Net和线性回归模型，生成分析结果。
3.  **临床辅助决策模块:** 医生主要使用的工作台，用于查看患者信息和AI分析结果。
4.  **智能康复随访模块:** 连接医生与患者，实现远程、自动化的术后管理。

### 3. 功能需求详述

#### 3.1. 模块一: 数据采集与标注平台 (主要用户: 医生, 科研人员)

| 需求ID | 功能点 | 详细描述 | 优先级 |
| :--- | :--- | :--- | :--- |
| **F-DM-01** | **创建新病例** | 医生可以为新患者创建唯一的病例档案。录入基本信息，如ID、年龄、联系方式、入院日期等。 | **高** |
| **F-DM-02** | **结构化信息录入** | 提供标准化的表单，用于录入患者的病理信息、基因信息、治疗方案、随访记录等文本数据。字段需遵循项目组制定的统一规范。 | **高** |
| **F-DM-03** | **影像数据上传** | 支持上传DICOM格式的MRI影像文件。系统后台自动解析DICOM元数据并与患者病例关联。 | **高** |
| **F-DM-04** | **AI辅助影像标注** | 1. 提供一个Web影像查看器，可逐层浏览MRI序列。<br>2. 提供基础的勾画工具（如画笔、多边形），让医生手动勾画肿瘤区域（ROI）。<br>3. **（核心）** 集成一个预训练的 **U-Net 模型**，提供“一键智能标注”功能。医生点击后，模型自动分割出可能的病灶区域，医生可在此基础上进行手动修正。 | **高** |
| **F-DM-05** | **数据质控** | 提供一个“提交审核”功能。标注完成的数据可由高级别专家进行二次审核，确保标注质量。 | **中** |

#### 3.2. 模块二: AI预后评估模型 (后端模块，无用户界面)

| 需求ID | 功能点 | 详细描述 | 优先级 |
| :--- | :--- | :--- | :--- |
| **F-AI-01** | **U-Net病灶分割** | - **输入:** 患者的MRI影像数据。<br>- **处理:** U-Net模型对影像进行像素级分割，识别出肿瘤区域。<br>- **输出:** 一个与原始影像大小相同的Mask（掩码）文件，其中病灶区域被标记出来。 | **高** |
| **F-AI-02** | **影像组学特征提取** | 从U-Net分割出的ROI中，自动提取一组基础的影像组学特征（如体积、形状、纹理等）。 | **中** |
| **F-AI-03** | **线性回归预后预测** | - **输入:** 患者的临床信息（如年龄、分期）、病理信息和影像组学特征。<br>- **处理:** 使用一个训练好的**线性回归模型**，对患者的预后关键指标（如复发风险、生存期等）进行打分或预测。<br>- **输出:** 具体的预测值或风险评分（例如：2年内复发风险评分 0.75）。 | **高** |

#### 3.3. 模块三: 临床辅助决策平台 (主要用户: 医生)

| 需求ID | 功能点 | 详细描述 | 优先级 |
| :--- | :--- | :--- | :--- |
| **F-CD-01** | **患者列表与检索** | 以列表形式展示所有患者，支持按姓名、ID、入院日期等条件进行搜索和排序。 | **高** |
| **F-CD-02** | **患者360°视图** | 点击患者后，进入详情页。该页面以标签页形式整合显示患者的所有信息：<br>1. **基本信息**<br>2. **临床与病理报告**<br>3. **影像查看器** (可与AI结果叠加显示)<br>4. **AI分析报告**<br>5. **随访记录** | **高** |
| **F-CD-03** | **AI分析结果可视化** | 在“AI分析报告”标签页中：<br>1. **影像结果:** 在MRI影像上叠加显示U-Net输出的病灶轮廓，并用不同颜色高亮。<br>2. **预测结果:** 清晰地展示线性回归模型输出的预后风险评分，并提供简单的参考范围说明（如：高风险/中风险/低风险）。 | **高** |

#### 3.4. 模块四: 智能康复随访系统 (主要用户: 医生, 患者)

| 需求ID | 功能点 | 详细描述 | 优先级 |
| :--- | :--- | :--- | :--- |
| **F-FU-01** | **创建随访计划** | 医生可为患者创建个性化的术后随访计划，设定随访节点（如术后1个月、3个月、6个月）和每个节点需要完成的任务（如填写量表、上传复查报告）。 | **高** |
| **F-FU-02** | **任务自动提醒** | 系统在随访节点前自动通过**微信/短信**向患者发送提醒，告知需要完成的随访任务。 | **高** |
| **F-FU-03** | **患者端应用** | 开发一个简单的**微信小程序或H5页面**，供患者使用：<br>1. 查看随访任务列表。<br>2. 在线填写健康状况调查问卷（如生活质量量表）。<br>3. 拍照上传最新的检查报告。 | **中** |
| **F-FU-04** | **随访数据看板** | 在医生的工作台中，提供一个随访看板，可以快速查看所有患者的随访状态（已完成/待办/逾期），并能快速浏览患者提交的随访内容。 | **高** |

### 4. 非功能性需求

| 类别 | 需求描述 |
| :--- | :--- |
| **性能** | - 影像数据加载时间：在标准网络环境下，MRI序列加载时间应小于5秒。<br>- AI模型推理时间：单次模型分析（U-Net + 线性回归）的后台处理时间应小于30秒。 |
| **安全性与隐私** | - **数据加密:** 所有患者敏感数据在传输和存储过程中必须加密。<br>- **数据脱敏:** 用于科研和模型训练的数据必须进行严格的脱敏处理，去除所有个人身份信息。<br>- **权限控制:** 严格区分不同用户角色（管理员、医生、研究员）的访问和操作权限。 |
| **易用性** | - 医生端界面设计应简洁直观，符合临床工作流程，最大程度减少学习成本。<br>- 患者端应用界面应字体大、操作简单，充分考虑老年用户的使用习惯。 |
| **兼容性** | 系统应兼容主流的Web浏览器（Chrome, Firefox, Edge最新版）。 |

### 5. 未来版本规划 (V2.0)
*   **模型升级:** 引入更复杂的深度学习模型（如Transformer）以提升预测精度。
*   **系统集成:** 探索与医院现有的HIS（医院信息系统）、PACS（影像归档和通信系统）进行数据接口对接，实现数据自动同步。
*   **移动端应用:** 为医生开发专用的移动App，方便随时随地查看患者信息和随访状态。
*   **多癌种支持:** 将平台架构扩展，使其未来可以支持其他癌种的研究与应用。
